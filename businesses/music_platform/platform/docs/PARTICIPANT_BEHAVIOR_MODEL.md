# PARTICIPANT_BEHAVIOR_MODEL

## 1. Purpose

本書は、参加者のユーザー行動を、機能要求ではなく実行単位で定義する正本である。

対象は `music_platform` における参加側の行動であり、

- 参加候補を見つける
- 参加できるか判断する
- 連絡する
- 担当を決める
- 準備する
- 当日を乗り切る
- 終了後に継続する

という一連の行動を、心理障壁と責務つきで書き起こす。

## 2. Design View

参加者の行動は、単なる「参加」ではなく次の5段階で進む。

1. Discover: 参加候補を見つける
2. Entry: 参加意思を固めて連絡する
3. Build: 担当と準備を整える
4. Run: 当日を安全に完了する
5. Review: 次回参加を判断する

## 3. Core Interpretation

参加者は「楽しみたい人」である前に、

- 失敗したくない
- 迷いたくない
- 直前で困りたくない
- 自分だけ負荷過多になりたくない

という心理を持つ。

したがって、参加者向けの体験設計は「情報提示」では足りず、
判断負荷を下げ、次の一歩を明確化し、不安を具体的に解消する必要がある。

## 4. Participant Behavior Model

### 4.1 Discover

#### 行動（Discover）

1. 候補イベントを探す
2. 自分に合う条件を見比べる
3. 参加できそうなイベントを3件以内に絞る
4. 主催者の信頼感を確認する
5. 初参加であれば不安が少ないかを確認する

#### 心理障壁（Discover）

1. 自分に合わないイベントを選ぶ不安
2. 初参加で浮くのではないかという不安
3. 条件が曖昧で判断できないストレス
4. 参加後に負荷が高すぎるのではという懸念

#### 責務（Discover）

1. 条件を比較しやすい単位で提示する
2. 初参加者向けの不安を隠さず明示する
3. 判断材料に根拠を添える
4. 候補を絞るための優先順位を示す

### 4.2 Entry

#### 行動（Entry）

1. 会への参加登録を行う
2. 参加可能パートを登録する
3. 成立支援通知の受信可否を選ぶ
4. 参加可能日程と担当上限を確定する
5. 参加締切までに会への参加登録を確定する
6. 曲別にエントリーする曲とパートを決める

#### 心理障壁（Entry）

1. 初回登録を完了する緊張
2. 参加登録と曲エントリーの違いが分からない不安
3. 何を先に決めるべきか分からない迷い
4. 自分が参加してよいか過度に技術面で悩む不安

#### 責務（Entry）

1. 会への参加登録と曲別エントリーを分けて提示する
2. 参加登録時に必要な参加可能パート、通知可否、担当上限を明示する
3. 技術審査ではなく、本人が準備して参加したいと思える範囲で選べることを明示する
4. 参加締切、曲別エントリー締切、推薦通知締切と未完了タスクを見える化する
5. 飛び入り演奏参加ではなく、事前登録者のみが当日演奏できることを明示する

### 4.3 Build

#### 行動（Build）

1. 担当曲の練習優先度を決める
2. 未成立曲の状況を確認する
3. 機械推薦の根拠、既存メンバー、自分の負荷を見て補完参加の可否を判断する
4. 承諾または辞退理由を選ぶ
5. 演奏候補曲を提案する
6. 当日までの準備を進める

#### 心理障壁（Build）

1. どこまで貢献すべきかの迷い
2. 自分が頑張りすぎることへの警戒
3. 未成立曲が多いときの焦り
4. 推薦を受けたときに断ってよいか分からない負担感
5. 候補曲の提案が採用されるか分からない不安

#### 責務（Build）

1. 優先度を明確にする
2. 補完の判断基準を示す
3. 準備負荷を分散する
4. 自分の役割と余力を確認できるようにする
5. 候補曲の提案が主催者専有ではないことを明示する

### 4.4 Run

#### 行動（Run）

1. 持ち物と到着手段を確認する
2. 当日動線を把握する
3. 遅刻や欠席時の連絡先を確認する
4. 現場変更に追従する
5. 演奏と参加を完了する

#### 心理障壁（Run）

1. 当日になってからの不安増大
2. 迷ったまま会場に行くことへの緊張
3. 変更連絡を見落とす不安
4. 実際の進行についていけない懸念

#### 責務（Run）

1. 当日チェックリストを短く提示する
2. 緊急連絡先をすぐ見つけられるようにする
3. 変更時の影響を明示する
4. 迷ったら次に何をすべきかを示す

### 4.5 Review

#### 行動（Review）

1. 体験を振り返る
2. 次回参加するか判断する
3. 次回候補を選ぶ
4. 継続参加の負荷感を見直す
5. 不安が解消できたか確認する

#### 心理障壁（Review）

1. 楽しかったが次回も行けるか不安
2. 継続すると負荷が増える懸念
3. 一度うまくいかないと離脱しやすい心理
4. 何を基準に次回を選ぶべきか曖昧な状態

#### 責務（Review）

1. 振り返りを行動に変換する
2. 次回候補を提示する
3. 継続判断の材料を短くまとめる
4. 離脱理由を言語化できるようにする

## 5. Participant-Specific Issues

1. 参加候補が多いと、比較軸が曖昧なまま離脱しやすい
2. 参加登録と曲別エントリーが混同されると、行動開始で止まりやすい
3. 未成立曲への貢献を人間関係や善意依存にすると、負荷偏りが起きやすい
4. 当日の変更連絡が見えないと、不安が急上昇する
5. 終了後の次回判断が曖昧だと継続率が落ちる
6. 候補曲提案を主催者だけの権利だと誤解すると、参加側の貢献機会が減る
7. 演奏技術を参加資格のように扱うと、セッション会/演奏会の参加機会が不必要に狭くなる
8. 参加締切が曖昧だと、当日飛び入り参加の誤解と運営破綻を招く

## 6. Modern Requirements

1. 参加候補は3件以内に収束させる
2. 参加登録時に参加可能パート、成立支援通知可否、担当上限を取得する
3. 参加締切・曲別エントリー締切・推薦通知締切・未返信・次の一手を常に明示する
4. 不確実性は隠さず表示する
5. 終了後の次回候補を即時提示する
6. 推薦は本人承諾なしにSongEntryを作成せず、辞退理由を能力評価へ変換しない
7. 当日画面は事前登録者のRun導線に限定し、飛び入り演奏参加導線を置かない
