# Asset Production Checklist

## Conclusion

Use this checklist before a Canonical Asset is approved and merged.

## Checklist

| ID | Criterion | Status |
|---|---|
| AP-01 | Asset has stable title | To verify |
| AP-02 | Asset type is defined | To verify |
| AP-03 | Objective is explicit | To verify |
| AP-04 | Scope and out of scope are explicit | To verify |
| AP-05 | Canonical body is complete | To verify |
| AP-06 | Reuse rules are defined | To verify |
| AP-07 | Risk notes are included | To verify |
| AP-08 | Quality Gate is complete | To verify |
| AP-09 | Registry entry is prepared | To verify |
| AP-10 | Source traceability exists | To verify |
| AP-11 | Related ADR/WBS is linked when applicable | To verify |
| AP-12 | Runtime implementation is not introduced | To verify |
| AP-13 | Approval bypass did not occur | To verify |

## Definition of Done

The asset may be treated as canonical only after Human approval and merge into `main`.
