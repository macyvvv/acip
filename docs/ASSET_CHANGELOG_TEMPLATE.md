# Asset Changelog Template

## Conclusion

Use this template to preserve the reasoning and version history of a Canonical Asset.

## Template

# `<asset_id>` Changelog

## Version History

| Version | Date | Change | Reason | Related PR | Approved By |
|---|---|---|---|---|---|
| 1.0.0 | YYYY-MM-DD | Initial canonical version | First approval | # | Human |

## Deprecation

If deprecated, record:

- deprecation date
- replacement asset id
- reason
- migration note
