# Category Taxonomy

## Conclusion

Categories provide stable high-level grouping for Knowledge Factory assets.

## Primary Categories

- company-os
- knowledge-factory
- canonical-asset-production
- asset-lifecycle
- asset-registry
- catalog-layer
- governance
- automation
- risk
- roi
- content-production
- platform-intelligence
- runtime-future

## Category Rules

- Use lowercase kebab-case.
- Prefer stable categories over temporary topics.
- Do not create a category for a one-off asset.
- New categories require review when they affect search or governance.
- Repository overrides conversation.

## Human Boundary

Category cleanup should be performed by Codex or future approved automation when rules are explicit.
