# ADR-0001: Stateless Orchestrator with Repository as Single Source of Truth

Version: v0.1
Status: Accepted

## Context

ACIPではChatGPT、Codex、Human、GitHubが協働する。

これまでの課題は以下である。

- Conversationに状態が残る
- Humanがコピー役になる
- Current Objectiveがぶれる
- Worker間の責務が曖昧になる
- 再開時に文脈復元が必要になる

すでにRepositoryには以下が存在する。

- platform/docs/current/CURRENT_STATE.md
- platform/docs/checkpoints/
- platform/.platform/system/prompts/codex/CHECKPOINT_WORKER.md
- platform/.platform/system/prompts/codex/ASSET_WORKER.md
- platform/.platform/system/SOP/CONTENT_PIPELINE.md
- platform/knowledge/draft/

これらを前提に、Orchestratorを導入する。

## Decision

ACIP OrchestratorはStatelessにする。

StateはRepositoryにのみ保持する。

Primary State Source:

platform/docs/current/CURRENT_STATE.md
