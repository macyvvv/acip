# 018 Asset Output Policy

## Conclusion

Outputs derived from Canonical Assets must preserve source meaning while adapting format, audience, and channel.

## Output Types

- Content Object
- Media Object
- Operational Asset
- Prompt
- Checklist
- SOP
- Decision memo
- Review memo

## Output Rules

- Derived outputs must reference source asset id.
- Derived outputs must not silently change source meaning.
- Platform-specific optimization is allowed only after preserving canonical intent.
- Revenue link should be stated when known.
- Risk notes must travel with derived outputs.
- Repository overrides conversation.
- Auto posting remains prohibited.
- Platform API integration remains prohibited.
- Runtime implementation remains out of scope.

## Required Output Metadata

- output_id
- source_asset_id
- output_type
- target audience
- target destination
- adaptation notes
- risk notes
- reuse status
- approval status
- version
