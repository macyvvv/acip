# 041 Status Reporting Policy

## Conclusion

Status reporting must be structured so Human receives decision-ready summaries, not raw operational burden.

## Required Status Sections

- conclusion
- current phase
- current objective
- completed work
- blocked items
- risks
- next action
- owner
- done condition
- Human decision required

## Human Boundary

Reports must not ask Human to perform routine investigation, file cleanup, validation, or metadata normalization.

If a task can be assigned to ChatGPT, Codex, script, GitHub Actions, or future approved automation, it should not be assigned to Human.

## Repository Rule

Repository overrides conversation.
