# REPOSITORY_CONVENTIONS

## Purpose

Define the repository naming convention for human-readable markdown files.

## Canonical Rule

Repository is the Single Source of Truth.

Conversation is not canonical.

## Markdown Documents

Rule:

`UPPER_SNAKE_CASE.md`

Examples:

- `CURRENT_STATE.md`
- `CONTENT_PIPELINE.md`
- `REPOSITORY_CONVENTIONS.md`
- `CHECKPOINT_M1_EP001.md`
- `CHECKPOINT_M1_ORCHESTRATOR_MVP.md`
- `MISSION.md`
- `ARCHITECTURE.md`
- `WBS.md`

## Python

Rule:

`lower_snake_case.py`

Examples:

- `state.py`
- `queue.py`
- `dispatcher.py`
- `worker.py`
- `result.py`
- `codex_worker.py`

## Directories

Rule:

`lower_snake_case/`

Examples:

- `basis/`
- `docs/`
- `knowledge/`
- `orchestrator/`
- `tests/`

## Never

- Spaces
- Mixed naming
- Temporary names
- Conversation as SSOT
