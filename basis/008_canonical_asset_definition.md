# 008 Canonical Asset Definition

## Conclusion

A Canonical Asset is a reusable, repository-governed knowledge artifact that can be adapted into platform-specific outputs without changing its source meaning.

## Purpose

Canonical Assets exist to create knowledge once, preserve it permanently, and reuse it across channels, products, campaigns, and future systems.

## Asset Types

| Type | Description | Primary Value |
|---|---|---|
| Knowledge Asset | Durable explanation, principle, playbook, or decision logic | Learning / Strategic |
| Content Object | Reusable content unit derived from a Knowledge Asset | Business / Brand |
| Media Object | Visual, audio, or video derivative of a Content Object | Distribution / Brand |
| Operational Asset | Checklist, SOP, template, prompt, or workflow document | Operational |

## Required Properties

A Canonical Asset must have:

- stable title
- asset type
- objective
- owner
- source context
- audience
- scope
- out of scope
- canonical body
- reuse rules
- risk notes
- quality gate status
- version

## Non-Canonical Materials

The following are not Canonical Assets by default:

- chat transcripts
- ad hoc notes
- platform posts
- drafts without review
- unapproved generated content
- temporary experiments

They may become Canonical Assets only after review and repository merge.

## Repository Rule

Only merged repository content is canonical.
