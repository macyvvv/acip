# 024 Tag Policy

## Conclusion

Tags must be stable, controlled, and reusable so assets remain searchable at scale.

## Tag Types

- domain
- function
- audience
- lifecycle
- risk
- value
- channel
- format
- dependency
- review

## Tag Rules

- Tags use lowercase kebab-case.
- Tags must be reusable across assets.
- Tags must not encode temporary opinions.
- Tags must not duplicate asset title.
- Tags must support search, grouping, or review.
- Deprecated tags remain listed until migrated.
- Repository overrides conversation.

## Examples

| Type | Example |
|---|---|
| domain | ai-native-company |
| function | asset-production |
| audience | founder |
| lifecycle | canonical |
| risk | high-risk |
| value | strategic-value |
| channel | x-twitter |
| format | checklist |
| dependency | adr-linked |
| review | needs-review |

## Automation Bias

Tag normalization should be performed by Codex or future approved automation, not Human, unless judgment is required.
