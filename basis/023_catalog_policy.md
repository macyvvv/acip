# 023 Catalog Policy

## Conclusion

ACIP must maintain a repository-governed Catalog Layer so Canonical Assets can be found, reused, combined, revised, and deprecated without relying on chat memory.

## Purpose

The Catalog Layer exists to maximize reuse and reduce duplicate asset production.

## Catalog Scope

The Catalog Layer includes:

- Knowledge Index
- Content Index
- Media Index
- Operational Index
- Deprecated Index
- Category Taxonomy
- Tag Standard
- Naming Standard
- Search Guideline
- Relationship Model

## Rules

- Repository overrides conversation.
- Every canonical asset should be discoverable through at least one index.
- Asset names must support search and reuse.
- Derived outputs must reference source asset id.
- Deprecated assets remain searchable.
- Human must not be assigned work that can be performed by ChatGPT, Codex, validation scripts, or future approved automation.
- Runtime implementation remains out of scope until explicitly approved.

## Human Boundary

Human is responsible for:

- Mission
- Approval
- Emergency Stop

Human should not be responsible for:

- routine indexing
- metadata normalization
- checklist filling
- mechanical validation
- duplicate detection
- registry hygiene
- catalog hygiene

Those should be delegated to ChatGPT, Codex, scripts, or future automation when feasible.
