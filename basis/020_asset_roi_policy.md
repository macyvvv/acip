# 020 Asset ROI Policy

## Conclusion

Canonical Asset Production must connect to Revenue as final KGI while also recognizing operational, learning, brand, and strategic value.

## Value Categories

| Category | Meaning |
|---|---|
| Business Value | Direct or indirect revenue creation |
| Operational Value | Execution speed, consistency, error reduction |
| Learning Value | Reusable wisdom and decision improvement |
| Strategic Value | Future optionality, defensibility, positioning |
| Brand Value | Trust, clarity, authority, audience formation |

## ROI Rule

Revenue is the final KGI.

However, assets may be approved before direct revenue if they improve execution efficiency, wisdom, maintainability, brand, or future optionality.

## Required ROI Fields

Each asset should record:

- expected value category
- expected revenue path
- production cost estimate
- reuse potential
- review cost
- maintenance cost
- measurement window
- leading indicator
- lagging indicator

## Measurement Windows

- 30 days
- 90 days
- 365 days
- lifetime

## Repository Rule

Repository overrides conversation.

ROI assumptions must be recorded in repository-governed files when they affect prioritization.
